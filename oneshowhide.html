1.背景

目前主流的条件渲染方式依据编译过程可区分为基于局部编译/卸载，及基于css切换。以Vue框架为例，v-if动态向DOM树内增删目标组件的相关DOM元素和事件监听及其子组件。v-show仅设置目标组件的DOM元素display样式属性。

2.问题

v-if，v-show各自的适用场景是什么？

目前博客上普遍的结论是“如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。但其中“非常频繁”与“很少改变”的指代较为模糊，因此需对该问题建立数学模型。

4.2 基于惰性渲染的组件显隐方案

4.2.1 简述

条件渲染通过首次采用v-if，之后采用v-show实现。

4.2.2 代码

示例中方案以组件实现。

xxxxxxxxxx
<template>
    <div>
        <div v-if="A" v-show="B">
            <slot></slot>
        </div>
    </div>
</template>
<script>
export default {
    props: {
        turn: {
            type: Boolean,
            default: false
        },
    },
    data () {
        return {
            isFirstTime: true,
            A: false,
            B: true
        };
    },
    watch: {
        turn: {
            handler: function () {
                if (this.isFirstTime) {
                    this.isFirstTime = false;
                    this.A = true;
                } else {
                    this.B = !this.B;
                }
            }
        }
    },
};
</script>
